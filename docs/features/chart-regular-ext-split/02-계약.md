# 02-계약 — 개별종목 차트 헤더 등락률 정규장/시간외 분리 (US 연장세션)

> 기능: US(해외) 종목 차트 카드/모달 헤더 등락률을 연장세션(프리·애프터·야간)에서 **통합(메인) + 정규장(보조 줄)** 2줄로 표기.
> 소유권: **API 계약 = backend-engineer**(아래) · **화면 명세 = frontend-engineer**(StockChart.vue `showUSExtBreakdown`)
> 작성: backend-engineer · 2026-07-16 · 상태: **동결(구현 완료)**

---

## 대상·범위

- **US(해외) 종목만.** KR 종목·지수는 현행 유지(단일 등락률, `regular_*`/`us_session`은 KR엔 null/무영향).
- 표시 결정: **메인 = 통합**, **보조 = 정규장**(보유종목 `HoldingsPanel` 2줄 표기와 동형).
- 시세 조회 로직(미국 현재가 3경로)은 **불변** — 응답 노출·등락률 계산만 확장.

## WS `getStockData` US 응답 페이로드 (App.vue `gridStockData` 소비)

| 필드 | 신규 | 의미 | 계산식 |
|---|---|---|---|
| `change_percent` | (기존, 의미변경) | **통합** 등락률(%) | `(현재가[시간외 포함] − 직전거래일 정규장 종가) / 직전거래일 정규장 종가 × 100`, 소수 2자리 |
| `change_amount` | (기존, 의미변경) | 통합 등락 금액 | `현재가 − 직전거래일 정규장 종가`, 소수 4자리 |
| `regular_change_percent` | ✅ | **정규장** 등락률(%). 시간외 무관·고정 | `(당일 정규장 종가 − 직전거래일 정규장 종가) / 직전거래일 정규장 종가 × 100`, 소수 2자리 |
| `regular_change_amount` | ✅ | 정규장 등락 금액 | `당일 정규장 종가 − 직전거래일 정규장 종가`, 소수 4자리 |
| `regular_close` | ✅(노출) | 당일 정규장 종가(정본 `yahoo_regular_close_{ticker}`) | 캐시값 그대로 |
| `us_session` | ✅ | 프론트 2줄 게이트 플래그 | `PRE`·`REGULAR`·`AFT`·`EXT_NIGHT`·`CLOSED` |
| `session` | (기존) | 한글 세션명(기존 소비처용) | `프리마켓`·`정규장`·`애프터마켓`·`주간거래`·`장마감` |

### `us_session` 코드 매핑 (getUsSession 한글 → 프론트 코드)

| 한글(getUsSession) | 코드(us_session) | 2줄 표시 |
|---|---|---|
| 프리마켓 | `PRE` | ✅ |
| 애프터마켓 | `AFT` | ✅ |
| 주간거래(야간) | `EXT_NIGHT` | ✅ |
| 정규장 | `REGULAR` | ❌(1줄, 통합=정규장) |
| 장마감 | `CLOSED` | ❌(1줄, 현행) |

프론트(StockChart)는 `regular_change_percent !== null && us_session ∈ {PRE, AFT, EXT_NIGHT}` 일 때만 정규장 보조 줄을 켠다.

## 세션별 동작 (기준가 = 직전거래일 정규장 종가)

| 세션 | 직전거래일 정규장 종가(기준) | change_percent(통합) | regular_change_percent |
|---|---|---|---|
| 정규장(REGULAR) | 어제 종가(candles[1]) | 오늘 정규장 등락 = **현행 불변** | null(1줄) |
| 애프터/야간-자정전(AFT/EXT_NIGHT) | 어제 종가(candles[1]) | 정규장+시간외 통합 | 오늘 정규장 등락 |
| 프리/야간-자정후(PRE/EXT_NIGHT) | 직전거래일 종가(candles[0]) | 시간외 통합 | 0%(당일 정규장 미개장) |
| 장마감(CLOSED) | candles[1] | **현행 불변**(직전거래일 하루 등락 유지) | null(1줄) |

- **정규장 중**: 통합 = 정규장이라 두 값이 같아짐(회귀 없음).
- **직전거래일 정규장 종가**: `calculate()`의 롤포워드 **이전** day-over-day 기준(candles[1]/candles[0]). `getUsPrevRegularClose()`가 별도 캐시(`toss_prev_regular_close_{ticker}`, **ET 자정 TTL**)로 산출 — 자정에 기준이 전진(candles[1]↔candles[0])하므로.
- **새 외부 조회 없음**: 이미 있던 /candles·regular_close 캐시 재사용.

## 에러/graceful

- 직전거래일 종가 cold → `calculate()` 폴백(1줄, 회귀 없음).
- `regular_close` cold → `regular_*` = null(연장세션이어도 1줄 degrade).
- 토스 토큰 실패(IP 미등록)·Yahoo 폴백 시엔 기존 폴백 체인 유지(`regular_*` 미제공 가능).
