### B. 문제 진단 (왜 좁은 폭에서 겹치는가)

7/14 subgrid 5열 개편(`grid-cols-[auto_auto_auto_auto_auto]`, 행별 `col-span-5 grid-cols-subgrid`)은 **넓은 폭 정렬은 개선**했으나 좁은 폭에서 4가지 구조 결함으로 겹친다:

| # | 결함 | 메커니즘 | 증상 |
|---|---|---|---|
| 1 | **`w-fit` 컨테이너가 `min-w-0` 부모 안에서 압축됨** | `w-fit`=`fit-content`는 가용폭이 좁으면 auto 트랙을 min-content 이하로 밀어붙인다. 부모 `flex-1 min-w-0`이 콘텐츠 미만 축소를 허용 → 트랙이 겹칠 때까지 수축. | 전 행 값이 서로 파고듦 |
| 2 | **단위($/원)가 별도 3열로 분리** | col2 값(`justify-self-end`)의 오른끝과 col3 단위(`justify-self-start`)의 왼끝이 `gap-x-1`(4px)만 두고 인접. 압축 시 값 토큰이 트랙을 넘쳐 단위 위로 올라탐. | **"원" 글자가 숫자와 포개짐**, "16000.00 $" 붙음 |
| 3 | **손익 열에 `ml-2` 마진 + `justify-self-end` 동시 적용** | 이미 우측 고정된 그리드 아이템에 좌마진을 더하면 트랙 폭이 부족할 때 아이템 좌변이 앞 열 영역으로 침범. | 손익액이 평가액과 겹침 |
| 4 | **부호·퍼센트 자리 미확보(가변 auto)** | "-1,173,920"(음수·8자리)와 "+880"의 폭 차가 큰데 트랙이 데이터별로만 커져, `gap-x-1`(4px)이 고밀도에서 시각적으로 부족. | 부호·% 가 이웃 값에 붙음 |

**핵심**: 5개 필드(라벨·값·단위·손익·손익률)를 ~260px 한 줄에 밀어 넣으면서, 컨테이너가 **콘텐츠보다 좁게 압축될 수 있게** 열려 있어 겹침이 발생한다. 겹침을 "구조적으로 불가능"하게 하려면 **(a) 컨테이너를 콘텐츠 미만으로 못 줄이게 하고 (b) 마진·별도 단위열을 제거**하면 된다.

### C. 리디자인 — 핵심 결정

1. **단위(원/$/원)를 숫자에서 분리해 별도 서브그리드 열로** (2026-07-15 개정 — 아래 "단위 열 재분리" 참조). 숫자 열=우측 정렬(tabular)·단위 열=좌측 정렬(고정 최소폭) → 단위의 좌측 edge가 한 열에 세로 정렬되고 숫자는 자릿수 우측 정렬. → 오와열 정렬 성립. (겹침은 §D의 `whitespace-nowrap`·`w-max`·부모 `overflow-x-auto` 로 구조적으로 차단되므로 단위를 분리해도 안전.) *(2026-07-15 8열 소수점 정렬 시도는 개발자 판단으로 롤백 — 6열 유지.)*
2. **그리드를 콘텐츠 미만으로 못 줄이게** → `w-fit` → **`w-max`**(항상 ≥ 콘텐츠 폭). 초과 시 부모의 기존 `overflow-x-auto`가 **가로 스크롤**로 흡수(압축 대신). → 결함 1 근절. (아래 계산상 427px 앱에서 스크롤 없이 들어맞음)
3. **모든 `ml-*` 마진 제거, 열 간격은 그리드 `gap-x-2`(8px)만** → 결함 3 근절.
4. **4열 subgrid + 부호·퍼센트 자리 `ch` 최소폭 확보** → 결함 4 근절. 7/14 subgrid 정렬 장점(3행 열 정합)은 유지·발전.
5. **색 규율**(DESIGN §5-2): 값=중립 앵커, 손익액·손익률만 신호색. 시장 구분은 FlagIcon(색 아닌 형상).

### D. 레이아웃 — 그리드 모델 (모든 폭 공통, 스택 3행) · **2026-07-15 단위 열 재분리 개정**

> 개정 이유: 단위를 숫자에 병합(`…+'원'`/`+'$'`)해 값 셀을 우측 정렬했더니, "원"(CJK ~1em)과 "$"(~0.5em)의 폭이 달라 우측 정렬해도 단위가 세로로 안 맞고 숫자-단위 경계가 들쭉날쭉("원$원 오와열 안 맞음"). → 단위를 다시 숫자에서 **분리해 별도 열**로 두고 숫자=우측·단위=좌측 정렬. 겹침(옛 결함 2)은 아래 `whitespace-nowrap`+`w-max`+부모 `overflow-x-auto`(floor)로 이미 차단돼 있어 단위를 분리해도 겹치지 않는다.
> *(2026-07-15 후속: 여기에 소수점까지 정수/소수 2열로 더 쪼갠 8열안을 시도했으나 개발자이 "정렬이 더 더럽다"고 판단해 롤백 — 아래 6열안이 확정본.)*

**컨테이너**(compact 루트):
`grid  grid-cols-[auto_auto_auto_auto_auto_auto]  gap-x-1.5 gap-y-1  w-max  items-center  text-xs  whitespace-nowrap`
- 6 트랙: `① 라벨/국기 · ② 값숫자 · ③ 값단위 · ④ 손익숫자 · ⑤ 손익단위 · ⑥ 손익률배지`
- `w-max`+`whitespace-nowrap`: 콘텐츠 폭 고정(압축·줄바꿈 불가). 부모 `overflow-x-auto`가 안전망(넘치면 가로 스크롤, 겹침 아님).
- `gap-x-1.5`(6px) 열 간격 · `gap-y-1`(4px) 행 간격. (단위 열 2개 추가분을 gap 축소로 흡수.)

**각 행**(환율/US/KR): `col-span-6  grid  grid-cols-subgrid  items-center`

셀 배치(좌→우):

| 트랙 | 셀 | 정렬 | 스타일 의도 |
|---|---|---|---|
| ① | 라벨/국기 | `justify-self-start` | 환율="환율"(`text-2xs` `text-base-content/50` 모노) · US/KR=`<FlagIcon>` |
| ② | 값 숫자 | `justify-self-end` | 중립 앵커 — `text-xs font-mono font-semibold text-base-content/80 tabular-nums` · `min-w-[7ch]` |
| ③ | 값 단위 | `justify-self-start` | 중립 — `text-xs font-mono font-semibold text-base-content/80` · `min-w-[1.5ch]` (원/$/원) |
| ④ | 손익 숫자 | `justify-self-end` | 신호색 — `text-xs font-mono font-semibold tabular-nums` + `text-up`/`text-down` · `min-w-[8ch]` |
| ⑤ | 손익 단위 | `justify-self-start` | 신호색 — `min-w-[1.5ch]` ($/원, 환율 델타는 단위 없음→빈 셀) |
| ⑥ | 손익률 | `justify-self-end` | 배지 — `text-2xs font-mono font-medium tabular-nums px-1.5 py-0.5 rounded-xs text-center min-w-[6.5ch]` + `bg-up-weak text-up`/`bg-down-weak text-down` |

- **정렬 원리**: 숫자 열(②·④)=`justify-self-end`+`tabular-nums`로 자릿수 우측 정렬 → 숫자 우측 edge 일치. 단위 열(③·⑤)=`justify-self-start`로 좌측 정렬 → 원/$/원 좌측 edge가 한 열에 세로 정렬(폭 다른 CJK "원" vs "$" 무관). subgrid 트랙이 3행 최대폭으로 커지므로 min-w 는 하한 보증.
- `min-w-[…ch]`: 모노 tabular 기준 최악값(음수·콤마·8자리) 자리 사전 확보 → 부호·% 가 절대 안 붙음.
- **환율 행 정합**: 환율 rate 숫자→②, 단위 "원"→③, `fxDelta`→④, ⑤ 빈 단위, `fxRate`배지→⑥ 로 넣어 US/KR 과 **완전히 같은 6열에 정렬**. `fxDelta===null`(prev_close 없음)이면 ④·⑥ **내용 비움(셀은 유지)** → subgrid 로 레이아웃 유지(graceful).

**ASCII 와이어(427px 앱, 스택 3행):**
```
┌①─────┬②─────────┬③──┬④──────────┬⑤─┬⑥────────┐
│ 환율  │  1,491.20 │원 │     -7.28 │   │[ -0.49%]│
│ [US] │ 15,924.00 │$  │  -2308.00 │$  │[-12.66%]│
```

- 폭 계산(11px JetBrains Mono ≈ 6.6px/char): 국기 20 + 값 60 + 값단위 12 + 손익 60 + 손익단위 12 + 배지 62 + gap(6×5=30) ≈ **256px**. 가용 ~250–280px 근처 → 대개 단일행, 초과 시 겹침 대신 가로 스크롤(안전).

### E. 반응형

- **기본(앱 427px 포함, 넓은 브라우저 전까지)**: 위 D 스택 3행. 이 한 레이아웃이 **모든 폭에서 겹침 불가**를 보증(핵심 요구).
- **선택 강화 — `≥ lg`(1024px, 넓은 브라우저)**: 3행을 가로 1줄 티커로 전환(`환율 │ US │ KR`, 사이 `border-l border-hairline pl-3 ml-3` 구분선, 각 클러스터 내부 순서 유지). 헤더 세로 절약 + 대시보드형 인상. **2차 패스로 분리 가능** — 코어(스택)만으로도 넓은 폭에서 좌측 정렬로 정상 동작하므로, 개발자이 브라우저 헤더 최적화를 요청할 때 착수.

### F. 정보 위계

1. **손익률 배지(④)** — 색+배경으로 가장 눈에 띄는 결론("얼마 벌었나/잃었나").
2. **손익액(③)** — 신호색 텍스트, 배지 다음.
3. **평가액(②)** — 중립색 앵커(맥락: "내 자산 규모"). 크기는 손익과 동일(12px)하되 **색으로 위계**(중립 vs 신호).
4. **라벨/국기(①)** — 최소·중립. 시장 구분은 색이 아닌 **국기 형상**.

크기를 키우지 않고 **색(중립/신호)과 배지 채움**으로만 위계를 만든다(DESIGN "크롬보다 데이터, 조용한 밀도").

### I. 접근성 (WCAG AA)

- **색 단독 금지**: 등락은 이미 **부호(+/−)가 값·배지에 항상 선행**(`formatProfitWon/USD/Rate`) → 색맹 대비 충족. *(선택: 배지 앞 `▲`/`▼` 캐럿 추가 가능하나 부호로 충분 — 넣으면 `aria-hidden`.)*
- **대비**: `text-up`/`text-down` on `base-100`, `text-base-content/80` 모두 DESIGN 검증 토큰(AA 이상). `/50` 라벨은 필수 정보 아님(중복 맥락)이라 허용.
- **스크린리더**: 국기 SVG 는 `aria-label`(FlagIcon 기존). 각 행에 `aria-label` 로 "미국 평가액 …, 손익 …, -12.24%" 요약 제공 권장(선택).
- **모션**: 플래시 260ms 는 `prefers-reduced-motion` 시 전역 style.css 가 duration 무효화(기존).

### K. 구현 매핑 (frontend-engineer용 · Tailwind 의도)

> **2026-07-15 재분리 개정** — 아래는 최종(단위 별도 열·6열) 기준. 이전 "단위 병합·4열" 매핑은 폐기. *(같은 날 8열 소수점 분리안도 시도했으나 개발자 판단으로 롤백 → 이 6열안이 확정.)*

**컨테이너**(compact 분기):
- `w-fit` → **`w-max`** (+ `whitespace-nowrap`)
- 5열/4열 안 → **`grid-cols-[auto_auto_auto_auto_auto_auto]`(6열)**
- `gap-x-1` → **`gap-x-1.5`** · `gap-y-0.5` → **`gap-y-1`**

**각 행**: `col-span-*` → **`col-span-6`** (subgrid 동일)

**셀 변경점** (6열 = ①라벨 ②값숫자 ③값단위 ④손익숫자 ⑤손익단위 ⑥배지):
1. **값·손익 셀을 숫자 span + 단위 span 2개로 분리** — 숫자 span=`justify-self-end tabular-nums min-w-[7ch(값)/8ch(손익)]`, 단위 span=`justify-self-start min-w-[1.5ch]`. 단위 열 좌측 정렬로 원/$/원 세로 정렬.
2. **숫자/단위 분리 헬퍼**(`utils/format.js`) — `usdParts`/`wonParts`(값·부호없음), `profitUsdParts`/`profitWonParts`(손익·부호항상) → `{ num, unit }` 반환, null→`{ num:'—', unit:'' }`. 템플릿에서 `.num`(숫자 span)·`.unit`(단위 span) 바인딩. 기존 `formatWon`/`formatProfitUSD`/`formatProfitWon`(단위 포함 문자열)은 **비-compact 분기·HoldingsPanel 이 그대로 사용** → 무변.
3. **플래시 tint**는 숫자 span에만(`rounded-xs px-1 py-0.5`). 손익 단위 span 은 `profitColorClass` 만(bg 없음).
4. **손익률 배지** — 자동 6열째. `justify-self-end min-w-[6.5ch] text-center …` + `profitColorClass` + weak bg (무변).
5. **환율 행** — rate숫자(②)·"원"(③)·fxDelta숫자(④)·빈 단위(⑤)·fxRate배지(⑥). `fxDelta===null`이면 ④·⑥ **내용만 비우고 셀은 렌더**(subgrid 순차 배치 유지 → US/KR 과 열 정합 보존). 단위열(⑤)은 환율 델타에 단위가 없으므로 항상 빈 셀.

**토큰**(DESIGN → Tailwind, 전부 기존 유틸): `text-up`·`text-down`·`bg-up-weak`·`bg-down-weak`·`text-base-content/80`·`rounded-xs`·`text-2xs`·`text-xs`·`tabular-nums`. **신규 토큰 없음.**

**포맷 함수**: `utils/format.js`에 `{num,unit}` 분리 헬퍼 4종(순수 함수, 단위 테스트). 기존 함수 시그니처·동작 무변.
