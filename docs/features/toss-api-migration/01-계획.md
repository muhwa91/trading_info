# 01-계획 — Yahoo/KIS → 토스증권 Open API 전면 교체 (시세 단일화)

> 주인: **tech-lead** · 읽는 이: 전원 · **참조 먼저**(이미 결정된 것 재분석 금지)
> 이미 끝난 토스 API 실측·코드 Explore 결과를 **근거로 설계만** 한다. 재조사 금지.

## 1. 목표 · 배경 · 범위

### 1.1 목표
미국·국내 **개별종목 현재가 / 차트 / 환율 / 종목마스터 / 세션판정**의 외부 시세 소스를 **토스증권 Open API 한 곳**으로 단일화한다. 현재 KIS·Yahoo로 **3중 중복**된 현재가 호출 경로(아래 1.4)를 토스 **배치호출 1곳**으로 통합한다.

### 1.2 배경 (왜 지금)
- KIS 미국 현재가 로직이 3곳에 중복돼, EXCD·세션 분기 변경 시 silent fail이 반복됨(2026-06-23 실사례, `CLAUDE.md` 주의사항에도 명시).
- 토스 `/prices`는 **콤마구분 최대 200종목/콜, 10 TPS** → 현재 40~50종목을 단일 콜로 커버, Guzzle Pool 동시성 관리·세션별 EXCD 폴백 로직 전체가 불필요해진다.
- 토스는 **세션 캘린더(`/market-calendar`)·종목마스터·계좌연동**까지 한 번에 제공 → 세션 판정의 직접 구현 부담도 줄일 수 있다.

### 1.3 범위에 포함 (토스로 교체)
| 영역 | 현재 | 교체 후 |
|---|---|---|
| 미국 개별 현재가 | KIS HHDFS00000300 (3곳) | 토스 `/prices` 배치 1곳 |
| 국내 개별 현재가 | KIS inquire-price (2곳) | 토스 `/prices` 배치 1곳 |
| 환율 USD→KRW | KIS t_rate(HHDFS76200200) → Yahoo 폴백 | 토스 `/exchange-rate` |
| 개별종목 차트(분/일봉) | Yahoo v8 chart | 토스 `/candles` |
| 종목마스터(이름·시장·우선주·ETF) | (분산/하드코딩) | 토스 `/stocks` |
| 세션 판정 | 자체 `MarketSessionService`(ET 하드코딩) | 토스 `/market-calendar` 위임(권고 ②, §2.6) |
| 보유 동기화(선택) | 수동 입력 위주 | 토스 `/accounts`·`/holdings`(옵션, §2.8) |

### 1.4 교체 대상 현재가 3중 중복 (실측 확인)
1. `app/Services/KisParallelPriceFetcher.php` 전체(L1~646) — **라이브 그리드 지배적 출처**. WS 사이클이 Guzzle Pool로 전종목 병렬 조회→`kis_realtime_price(_us)_{ticker}` 캐시.
2. `app/Http/Controllers/StockController.php::fetchOverseasPriceFromKis`(L2276~) + `fetchDomesticPriceFromKis`(L1640~) — REST 단건/모달 경로.
3. `app/Services/Quote/KisOverseasQuoteProvider.php`(L45~493) + `KisDomesticQuoteProvider.php`(L19~185) — 포트폴리오 평가가(`PriceService`→`DashboardController`).

### 1.5 범위에서 제외 (격리·잔존)
- **🔴 지수 = 토스에 없음** → **Yahoo/KIS 현행 유지**(개발자 지시 확정). 대상: 나스닥100선물(`NQ=F`, Yahoo), 코스피종합(`KIS 0001`), 코스피야간선물(`KOSPI_NIGHT`, NQ 합성), `KOSPI200`(KIS).
  - 잔존 코드: `StockController::getIndices`(L500~), `getKospiIndexData`(L1467~), `getKOSPINightChartData`(L1247~), `kisIndexRequest`(L1333~), `kisIndexChange`(L1374~), `parseYahooFinanceChart`(L576~).
- 따라서 **KIS는 "지수 코스피용"만 잔존**, Yahoo는 "지수(NQ=F)용"만 잔존. 개별종목·환율·세션에서는 둘 다 제거.
- **주문 API 미사용** — 조회 전용·주문 금지 원칙 유지(`/accounts`·`/holdings`는 읽기만).

### 1.6 KIS 통째 제거 가능 여부 (판정)
**불가 — 부분 제거.** 코스피 종합지수(0001)·KOSPI200·야간선물이 KIS 의존이라 `kisIndexRequest`/토큰 발급 경로는 **남겨야 한다**. 단, 개별종목 현재가용 KIS 코드(§1.4의 1·2·3)와 환율 KIS 경로는 **완전 제거 대상**. → KIS 자격증명(`KIS_APP_KEY` 등)은 `.env`에 **지수 전용으로만 잔존**.

## 2. 아키텍처 설계

### 2.1 핵심 원칙 — 토스 게이트웨이 단일화
신규 `App\Services\Toss\` 네임스페이스에 **토스 API 클라이언트 1개 + 배치 시세 페처 1개**를 두고, 기존 3중 경로가 모두 이 한 곳을 바라보게 한다.

```
[WS 사이클 / REST / 포트폴리오 평가]
        │  (셋 다 동일 진입)
        ▼
  TossPriceFetcher (배치 200/콜, 10 TPS)  ──┐
  TossCandleProvider (/candles)            ├─→ TossApiClient (토큰 24h 캐싱 + rate-limit 가드)
  TossFxProvider (/exchange-rate)          ├─→        ▼
  TossStockMaster (/stocks)                ┘   POST /oauth2/token (client_credentials)
  TossSessionProvider (/market-calendar)
        │
        ▼ (지수만 분기)
  [기존 Yahoo NQ=F · KIS 0001 경로 — 손대지 않음]
```

### 2.3 심볼 매핑 규칙 (`TossSymbolMapper` 단일 소유)
| 입력(앱 내부) | 토스 `symbols=` | 비고 |
|---|---|---|
| `TSLA`, `MU`, `SOXL`, `TQQQ` | `TSLA`(순수 티커) | 미국. 콜론 포맷(`US:TSLA`) **불가**(패턴 `^[A-Za-z0-9.,\-]+$`). |
| `005930.KS`, `005935.KS` | `005930`, `005935` | **`.KS`/`.KQ` 접미사 제거**. 우선주(005935)도 동일. |
| `0167A0` (신형 영숫자) | `0167A0` | 그대로 통과(토스 OK 실측). `StockController` L124의 `0167AO→0167A0` 오타교정 유지. |
| `NQ=F`, `^KS200`, `KOSPI200`, `KOSPI_NIGHT` | **(토스 안 거침)** | **지수 → Yahoo/KIS 경로로 분기**(skip 목록). |

- 분류 정규식은 기존 것 재사용: 국내=`/(\.KS|\.KQ)$/i` 또는 `/^\d{4}[0-9A-Z]{2}$/` 또는 `/^\d+$/`(`KisParallelPriceFetcher` L87~91). 그 외=미국. **이 판정 로직을 `TossSymbolMapper`로 일원화**(현재 동일 정규식이 3~4곳 산재).
- skip 목록(`KisParallelPriceFetcher` L75): `['NQ=F','^KS200','USDKRW=X','KOSPI_NIGHT','KOSPI200']` → 환율은 `TossFxProvider`가 전담하므로 토스 배치에서도 동일 skip 유지, 지수는 분기.

### 2.4 등락 계산 설계 (`/prices`엔 등락 없음)
토스 `/prices` 응답 = `{symbol, timestamp, lastPrice, currency}` **만**. 등락액·등락률·전일종가·거래량 없음. → 현재 KIS가 주던 `change_amount`/`change_percent`/`regular_close`를 **앱이 직접 계산**해야 한다.

- **전일종가(prevClose) 캐싱**: `TossChangeCalculator`가 종목별 `/candles?interval=1d&count=2`로 **직전 완료봉 종가**를 가져와 `toss_prev_close_{symbol}` 캐시(TTL=당일 자정까지, KST/ET 시장별). KIS의 `kis_last_successful_*` 폴백 패턴과 동형.
- **등락 = (lastPrice − prevClose)**, **률 = 등락/prevClose×100**. 부호는 차액 부호로 자동(KIS의 `sign 4/5` 분기 불필요해짐 → 단순화).
- **미국 `regular_close`(정규장 보조줄)**: 현재 `KisOverseasQuoteProvider`가 Yahoo `regularMarketPrice`로 채우던 값(L380, L450~492). 토스에선 **`/candles?interval=1d`의 최근 완료 정규장 봉 종가**로 대체. 애프터/야간엔 "오늘 종가", 프리마켓엔 "어제 종가"가 자동으로 잡히도록 봉 시각 기준 판정.
- prevClose 캔들 조회는 **종목당 1일 1~2회면 충분**(자정 캐시) → `/candles` 5 TPS 부담 미미.

### 2.5 폴링 / 캐시 재설계 (배치 200·10 TPS·토큰 24h)
- **WS 3초 사이클 유지**(`WebSocketAgentServer` L121~123). 단, 현재 종목당 1요청(Guzzle Pool 동시성 8)을 **`/prices` 단일 배치콜**로 대체 → 40~50종목이 1콜. 10 TPS 한도 대비 사이클당 1~2콜이라 여유 충분.
- **배치 청크**: 종목 200 초과 시 `array_chunk(195)`로 분할(여유 마진). 현재 규모(40~50)에선 단일 콜.
- **캐시 키 하위호환**: `TossPriceFetcher`가 결과를 **기존 키 그대로** 채운다 — `kis_realtime_price_{ticker}`(국내)·`kis_realtime_price_us_{ticker}`(미국), 폴백 `kis_last_successful_*`. → `StockController::getStockData`(L285~302, L394~416)·프론트 무변경. *(키 이름의 `kis_` 접두는 추후 정리 항목으로 남김 — 이번엔 동작 호환 우선.)*
- **TTL**: 현재가 캐시 8초 유지(`PRICE_CACHE_TTL` L48), 토큰 `toss_access_token` TTL **82800초(23h)**(만료 86399 대비 마진). 환율 5분(`FX_STALE_SECONDS` L33 유지). prevClose 자정.
- **rate-limit 가드**: `MARKET_DATA 10 TPS`·`MARKET_DATA_CHART 5 TPS`를 `TossApiClient`에서 토큰버킷/최소간격으로 방어(KIS의 `usleep 100ms`·동시성8 가드 대체).

### 2.6 세션 라벨링 — 권고: **MarketSessionService 유지(① 기본), 캘린더 보강(②)**
- 현재 세션 SSOT = `app/Services/MarketSessionService.php`(`getUsSession` L30, `getKrSession` L77). `StockController`(L2216,L2253)·`KisParallelPriceFetcher`(L620~645)·`KisOverseasQuoteProvider`(L127~173)가 각자 ET 하드코딩으로 중복 판정 중.
- **권고 ①(기본·이번 범위)**: 토스 전환으로 **EXCD 세션 분기 자체가 사라진다**(토스 `/prices`는 거래소 선택 불필요·장외 자동 반영). 따라서 세션은 **라벨 표시·휴장 판정 용도로만** 남고, `MarketSessionService`를 **단일 SSOT로 유지**(중복 ET 판정 3곳은 토스 코드 제거와 함께 자연 소멸).
- **권고 ②(선택·후속)**: 휴장일·공휴일 정확도를 위해 `isUsMarketTradingToday`(Yahoo SPY 의존, `KisOverseasQuoteProvider` L181~220)를 **토스 `/market-calendar/US·KR`로 교체**. → Yahoo SPY 의존 제거. 단 이번 Phase 범위 밖으로 두고 **개발자 결정**(§6).

### 2.7 지수 격리 경계 (잔존 코드 무영향 보장)
- 지수 4종(`NQ=F`/`KOSPI200`/`KOSPI_NIGHT`/`^KS200`)은 `getStockData` 진입부(L131~262)에서 **토스 분기 이전에** 가로채진다 → 토스 페처는 skip 목록으로 절대 닿지 않음.
- `getIndices`(L500~)·`kisIndexRequest`(L1333~)는 헤더 인디케이터·메인 차트 전용 경로로 **그대로 둔다**.
- **경계 테스트**(qa): 토스 전환 후에도 지수 4종 응답이 교체 전과 동일한지 회귀 확인(§5 베이스라인).

### 2.8 계좌 자동연동 — 권고: **이번 범위 제외(옵션 보류)**
- 토스 `/accounts`·`/holdings`(헤더 `X-Tossinvest-Account`)로 실보유 자동 동기화 가능(개발자 실계좌 조회됨, 현재 보유 0).
- **권고**: 조회 전용 원칙엔 부합하나, ① 현재 토스계좌 보유 0이라 효용 없음, ② 보유는 이미 수동입력+`source(manual/synced)` 안전장치 존재, ③ 시세 교체와 **관심사 분리**. → **이번 대형 교체에선 제외**, 별도 소기능으로 후속. 채택 여부 §6에서 개발자 결정.

## 6. 개발자 확인 / 결정 사항

1. **세션 소스** — 권고: ① `MarketSessionService` 유지(이번 범위), ② `/market-calendar` 휴장일 교체는 **후속**. ②를 이번에 포함할지? (기본=후속)
2. **계좌 자동연동(`/holdings`)** — 권고: **이번 범위 제외**(보유 0·관심사 분리). 별도 소기능으로 둘지, 지금 포함할지?
3. **지수 처리** — **Yahoo/KIS 유지 확정**(지시대로). 추가로 KIS 키를 지수 전용으로 `.env` 잔존시키는 데 이견 없는지 확인.
4. **`regular_close` 산출 변경 허용** — Yahoo `regularMarketPrice` → 토스 `/candles 1d` 봉으로 변경(평가손익 보조줄). 베이스라인 허용오차 기준 승인.
5. **Phase 0 게이트** — 주간거래/애프터 실측 전까지 **미국 현재가 전환(Phase 4) 보류**에 동의하는지.
6. **캐시 키 명칭** — `kis_realtime_price_*`를 호환 위해 **그대로 유지**(이번엔 명칭 변경 안 함)에 동의하는지.
