01

개요 & 목표

무엇: 한국 주식·미국 주식·ETF를 한 곳에 모아, 보유 종목의 평가손익과 즐겨찾기 종목 시세를 원화 기준으로 한 화면에서 확인하는 개인용 웹앱.

목적: ① 개발자 본인이 미장+한국장 투자 현황을 통합 확인 ② 취업 포트폴리오(기획→배포까지 1인 완주).

1차 사용자: 개발자 본인 1명. 처음엔 로컬에서 본인만 사용 → 완성 후 배포(이때 멀티유저·로그인 추가).

핵심 가치: 흩어진 미장/한국장을 원화로 통합 + 환율 변동까지 반영한 실손익을 폴링으로 준실시간 표시.

02

범위 (In / Out)

가장 중요한 결정: 이 앱은 주문(매수/매도)을 절대 하지 않는다. KIS API는 조회(읽기) 권한만 사용.

In Scope (이번에 만든다)

  • 보유 종목 평가손익 (원화 환산, 환율 손익 포함)
  • 미장 + 한국장 + ETF 통합 (한 테이블 type으로 구분)
  • 즐겨찾기(watchlist) + 종목 검색
  • 폴링 기반 준실시간 시세·손익 갱신 (5~15초)
  • 프리마켓 포함 + 기준 선택 라디오 (전일종가/프리/정규장)
  • 시세 캐싱 (호출 한도 절약)
  • 차트 (Lightweight Charts, 보유는 평단선)
  • (선택) 배당 기록·세금 정리

Out of Scope (만들지 않는다)

  • 주문/매매 실행 — 읽기 전용, 절대 주문 안 함
  • 거래내역(trades) 테이블 — 확인용엔 불필요, 제외
  • 실현손익 계산 (거래내역 없으므로)
  • 투자 추천·조언·신호 — 자문업 회피, 계산·표시만
  • 로그인/회원가입 — 배포 단계까지 보류
  • 자동 잔고 동기화 — 1차는 수동 입력 (나중 옵션)
  • WebSocket 틱 실시간 — 1차 제외 (나중 종목상세 옵션)

06

손익 계산 로직

한국장은 원화 그대로, 미장은 환율을 곱해 원화 환산. 미장의 진짜 원화 손익은 주가 손익 + 환율 손익이 합쳐진 값 — 이 앱은 둘을 분리해서 보여준다. 그래서 현재 환율(fxNow, 장중 폴링)과 매입 환율(avg_fx_rate, portfolio에 저장) 두 개를 쓴다.

// 종목 1개의 원화 평가/손익 (선택된 session 가격 + 장중 환율)
function evaluate(holding, price, fxNow) {
  const isUSD = holding.stock.currency === 'USD';
  const fxBuy = isUSD ? holding.avg_fx_rate : 1;  // 매입 시 환율(KR=1)
  const fxCur = isUSD ? fxNow              : 1;  // 현재 환율(장중 폴링)
  const qty = holding.quantity, avg = holding.average_price;

  const marketValueKRW = price * qty * fxCur;     // 평가액(원화)
  const costKRW        = avg   * qty * fxBuy;     // 원화 매입원가(고정)
  const profitKRW      = marketValueKRW - costKRW; // 총 평가손익(원화)

  // ── 손익 분리 (priceProfit + fxProfit = profitKRW, 교차항 없음) ──
  const priceProfitKRW = (price - avg)  * qty * fxCur; // 주가손익(현재환율 환산)
  const fxProfitKRW    = (fxCur - fxBuy) * avg * qty;  // 환율손익(매입원가 기준)

  const profitRate = (price - avg) / avg;          // 수익률(통화무관)
  return { marketValueKRW, profitKRW, priceProfitKRW, fxProfitKRW, profitRate };
}
// 포트폴리오 합계 = 모든 보유 종목 evaluate() 합산 (각 항목별로 합산)

분해 검증: priceProfitKRW + fxProfitKRW = profitKRW 가 항상 성립(교차항 없음). 한국장은 fxBuy=fxCur=1이라 환율손익=0, 주가손익=총손익.

주의: average_price와 price는 같은 통화(종목 원래 통화)여야 한다. 수익률(%)은 통화 무관(환율 약분)하지만, 원화 평가액·손익은 환율을 곱한다.

07

세션 & 기준 선택 라디오

자동 판단 대신 사용자가 라디오로 손익 기준을 직접 선택한다. 손익 공식은 동일하고 "현재가"에 어느 세션 가격을 넣느냐만 바뀐다.

전일 종가 기준 prev close
프리마켓 기준 session=pre
정규장 기준 session=regular · 장중에만

각 선택지 → 사용할 가격:

· 전일 종가 = prices에서 어제(직전 거래일) session=regular의 마지막 가격. 고정값(프리마켓 변동 제외하고 보고 싶을 때).

· 프리마켓 = 오늘 session=pre 최신 가격. 폴링으로 갱신.

· 정규장 = 오늘 session=regular 최신 가격. 장중에만 활성, 그 외 비활성(회색).

혼합 표시: 미장/한국장 거래시간이 다르므로 각 종목 옆에 현재 적용된 세션 배지(예: PRE/REG/전일)를 표시해 헷갈림 방지. 프리마켓은 거래량이 적어 변동이 크니 "참고용" 안내 표기.

08

폴링 전략

구조: Vue가 일정 간격(5~15초)으로 백엔드에 "최신 시세 줘" 요청 → Laravel이 캐시 신선하면 캐시, 오래됐으면 KIS 호출 후 갱신해 응답 → 화면 손익 숫자만 갱신.

// Vue (대시보드)
onMounted(() => { timer = setInterval(fetchPrices, 10000); }); // 10초
onUnmounted(() => clearInterval(timer));
document.addEventListener('visibilitychange', () => {
  document.hidden ? clearInterval(timer) : start(); // 백그라운드면 멈춤
});

// Laravel (시세 엔드포인트)
$ids = union(portfolioStockIds, watchlistStockIds); // 합집합·중복제거
foreach ($ids as $id) {
  if (cacheStale($id, $session)) $this->kis->fetchPrice($id, $session); // 부족분만
}
if (cacheStale('fx:USDKRW', minutes:5)) $this->fx->fetchUsdKrw(); // 환율도 폴링, 단 긴 캐시
return latestPrices($ids, $session) + latestFxRate();

규칙: ① 표시 종목 = 보유 ∪ 관심 합집합으로 한 번에. ② 시장 닫혀 있으면 폴링 멈추고 마지막 가격 유지(헛호출 방지). ③ 보유 종목 묶어서 호출, KIS rate limit 준수. ④ 탭이 백그라운드면 폴링 정지. ⑤ 환율도 같은 폴링 응답에 포함하되 시세보다 긴 캐시(수 분)로 — 환율은 천천히 움직이고 소스 한도를 아끼되, 미장 원화 손익이 종일 고정되지 않게.

부드러운 봉(캔들) 생성 — 폴링인데 끊기지 않게

폴링(5~15초)을 그대로 그리면 형성 중 봉이 매 폴링마다 새로 생기거나 종가가 계단식으로 점프한다. 해결은 ①봉 제자리 갱신 + ②시각 보간의 분리.

분봉 단위(timeframe): 사용자가 선택 — 1분 · 3분 · 5분 · 10분 · 1시간 · 1일. 선택값이 봉 버킷 크기가 되고, 형성 중 봉 갱신·이징은 단위와 무관하게 동일하게 동작한다.

3분 3m
5분 5m
10분 10m
1시간 1h

1

틱 → 시간버킷에 누적

폴링 가격을 선택된 분봉 버킷(1m~1d)으로 묶음. 같은 버킷이면 새 봉 X — 기존 봉의 고가/저가/종가만 키운다(series.update(), 같은 time). 버킷이 넘어갈 때만 새 봉 시작.

2

종가를 이징으로 글라이드

직전 표시 종가 → 새 종가를 requestAnimationFrame easeOut으로 수백 ms 보간. 폴링 빈도와 시각 부드러움을 분리 → rate limit 안 늘리고도 매끄럽게.

3

정직성 가드

보간은 형성 중 봉의 종가 표시에만. 고가/저가는 실제 도달값 즉시 반영(보간이 실제 범위를 넘지 않음). 버킷 마감 시 실제 OHLC로 확정. "표시는 부드럽게, 데이터는 실제값."

4

과거 봉은 KIS 분봉/일봉 조회

초기 로드는 선택 단위로 setData(), 이후 형성 중 봉만 틱으로 갱신. 출처: 1분=KIS 분봉 직접 · 3/5/10분·1시간=1분봉 집계 · 1일=일봉 API. prices는 손익용 최신-틱 캐시로 역할 분리(과거 봉 저장은 선택 price_candles).

// 1) 형성 중 봉: 선택된 분봉 버킷으로 묶어 제자리 갱신, 넘으면 새 봉
const TF = { '1m':60, '3m':180, '5m':300, '10m':600, '1h':3600, '1d':86400 }; // 초
let tf = '1m';                                  // 선택된 분봉(라디오)
const bucketOf = ts => Math.floor(ts/1000 / TF[tf]) * TF[tf]; // LWC 초 단위 버킷 시작
let cur = null;                                   // 형성 중 봉 {time,open,high,low,close}

function onTick(price, ts) {
  const t = bucketOf(ts);
  if (!cur || cur.time !== t)
    cur = { time:t, open:price, high:price, low:price, close:price }; // 새 봉
  else {
    cur.high = Math.max(cur.high, price);
    cur.low  = Math.min(cur.low,  price);
    cur.close = price;                       // 실제 종가(데이터)
  }
  animateClose(cur);                         // 표시는 부드럽게
}

// 2) rAF 이징: 직전 표시 종가 → 실제 종가로 글라이드 (고가/저가는 실제값 유지)
let shown = null;
function animateClose(bar) {
  const target = bar.close, start = shown ?? target, t0 = performance.now(), D = 600;
  function frame(now) {
    const p = Math.min((now - t0) / D, 1), e = 1 - Math.pow(1 - p, 3); // easeOutCubic
    shown = start + (target - start) * e;
    series.update({ ...bar, close: shown });  // time 동일 → 같은 봉 갱신
    if (p < 1) requestAnimationFrame(frame);
  }
  requestAnimationFrame(frame);
}

왜 안전한가: 고가/저가를 실제 가격으로 먼저 갱신하므로, 그 사이를 흐르는 shown 종가는 항상 봉의 실제 범위 안에 머문다(가짜 고저 생성 없음). 시간 점프(prefers-reduced-motion)면 D=0으로 보간 끄고 즉시 반영.

분봉 전환 시: cur·shown 초기화 → 해당 단위 과거 봉 재조회 후 setData() → 폴링 틱은 새 tf 버킷으로 누적. 1일봉은 time을 일자(자정) 기준으로 맞춘다.

11

KIS API 연동 노트

권한: 조회(읽기)만 사용. 주문 관련 엔드포인트는 호출하지 않음.

토큰: 접근 토큰은 발급 후 약 24시간 유효 → 캐시(DB/cache)에 TTL과 함께 저장하고 재사용, 만료 임박 시 갱신. 매 요청 재발급 금지(발급 한도).

환경 분리: 모의투자 / 실전 도메인·키가 다름. 개발·테스트는 모의에서, 실계좌는 검증 후. 키는 .env에만.

국내 vs 해외: 엔드포인트·필드 다름. 앱 내부에 QuoteProvider 인터페이스로 추상화(국내/해외 구현 분리) → 나머지 로직은 시장 분기 불필요.

정규 vs 시간외: 프리마켓·애프터마켓은 해외 현재가 조회의 "시간구분" 파라미터로 구분 조회 → prices.session에 매핑 저장. (정확한 파라미터명은 KIS 문서 확인.)

Rate limit: 보유·관심 합집합을 묶어 호출, 캐싱(1~5분)으로 화면 재오픈 시 재호출 방지.

잔고(나중): 잔고 조회로 보유 종목 자동 동기화 가능 → 그래도 수동입력 병행 유지(source).

13

제약 · 비목표 · 주의

  • 읽기 전용. 주문/매매 실행 기능을 만들지 않는다. KIS 조회 권한만 사용.
  • 투자 자문 아님. 매수/매도 추천·신호·조언 표시 금지. 계산·표시만(자문업 회피). "투자 판단은 본인 책임" 면책 표기.
  • 프리마켓은 참고용. 거래량 적어 호가 불안정·변동 큼. 정규장 시작 시 가격이 달라짐을 안내.
  • 장 마감 시 정지. 해당 시장 폐장 시 가격 고정 → 폴링 멈추고 마지막 값 표시.
  • 봉은 표시만 보간. 차트는 부드럽게 흐르게 그리되 데이터는 실제 OHLC. 보간이 실제 고가/저가 범위를 넘지 않음(가짜 시세 생성 금지).
  • 보안. KIS 키·계좌정보는 .env에만. DB·코드·저장소에 평문 금지.
  • API 비의존 안전장치. 장애/만료 대비 수동입력 유지(source 컬럼).
  • 데이터 라이선스. 시세는 본인 계좌 확인 용도. 외부 재배포·상업 제공은 별도 검토.
